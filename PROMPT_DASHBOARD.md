## 🚀 Dashboard Workflow (15-Line Summary)
You are building the clinical command center for doctors managing chronic diseases.
1. **Onboarding:** You register patients via `POST /onboard` including identity and initial vitals.
2. **AI Analysis:** The backend immediately returns a clinical summary and a Darija welcome message.
3. **Triage:** The home screen is the "Risk Queue" (`GET /risk-queue`), auto-sorted by clinical danger.
4. **Visibility:** You see every assessment generated by the patient's mobile app in real-time.
5. **Trends:** You visualize Blood Pressure and Glucose trends using Recharts on the patient detail page.
6. **Drift Detection:** You receive 🔴 alerts if a patient's medication adherence drops significantly.
7. **Intelligence:** You use `POST /doctor/chat` to ask the AI natural language questions about history.
8. **Documentation:** You generate professional PDF reports (`POST /reports/generate`) for medical records.
9. **Centralization:** All history, chats, and tests are stored in Supabase, accessible from any device.
10. **Triage UI:** Doctors can sort, filter, and prioritize patients without manual data entry.
11. **Clinical Focus:** You provide the doctor with the "Suggested Focus" extracted by AI.
12. **Alerting:** You highlight patients who haven't checked in for over 7 days.
13. **Family Link:** You manage family contact info and access permissions for emergency alerts.
14. **System Health:** You monitor backend and AI service status via a live header indicator.
15. **Data Ownership:** You "own" the patient profile while the mobile app "owns" the daily chats.

## 🤝 Teammate Role (5-Line Summary)
The mobile app developer builds a Flutter app for patients. Patients use it to chat with Hela AI and record their vitals. Every message they send creates a record in the `patient_assessments` table that you read. You provide the profile data they see; they provide the clinical data you analyze.

## 🔌 REAL API ENDPOINTS

### 1. `GET /api/v1/health`
**Usage:** Dashboard header status indicator.
**Returns:** `{ "status": "healthy", "services": { "database": "connected", "gemini": "initialized" } }`

### 2. `POST /api/v1/patients/onboard`
**Usage:** Add Patient Form. Creates the profile in Supabase.
**Request:**
```json
{
  "profile": {
    "name": "string", "age": 65, "gender": "F", "phone": "string", "address": "string",
    "family_contact_name": "string", "family_access_granted": true
  },
  "initial_vitals": { "systolic_bp": 150, "fasting_glucose": 140, "bmi": 27.5 },
  "is_import": false
}
```
**Returns:** `patient_id`, `initial_risk` (HIGH/MOD/LOW), and `ai_analysis` (summary + suggested_focus).

### 3. `GET /api/v1/patients/risk-queue`
**Usage:** Main Triage Table.
**Returns:** Array of `{ patient_id, risk_score, predicted_risk_level (0,1,2), symptoms (latest note) }`.
**Note:** Sort by `predicted_risk_level` descending (2=HIGH, 0=LOW).

### 4. `GET /api/v1/patient/{id}/profile`
**Usage:** Fetching patient name/info for the table rows and detail page.

### 5. `GET /api/v1/patient/{id}/history?days=30`
**Usage:** Render BP and Glucose charts.
**Returns:** Array of `{ date, systolic, diastolic, glucose, risk, summary }`.

### 6. `GET /api/v1/patient/{id}/check-drift`
**Usage:** Adherence alert panel on detail page.
**Returns:** `long_term_adherence`, `short_term_adherence`, `adherence_drop`, `trigger_notification` (bool).

### 7. `POST /api/v1/doctor/chat`
**Usage:** Natural language query box.
**Request:** `{ "patient_id": "uuid", "question": "Any chest pain reported?" }`
**Returns:** `{ "answer": "string", "history_analyzed": 50 }`

### 8. `POST /api/v1/reports/generate`
**Usage:** PDF Download button.
**Returns:** `application/pdf` binary stream.

## 🏗️ TECHNICAL STRUCTURE

### TypeScript Models
```typescript
export interface RiskItem {
  patient_id: string;
  predicted_risk_level: 0 | 1 | 2; // 0=Low, 1=Mod, 2=High
  risk_score: number;
  symptoms: string;
}

export interface PatientHistory {
  date: string;
  systolic: number;
  diastolic: number;
  glucose: number;
  risk: "LOW" | "MODERATE" | "HIGH";
}
```

- `/src/components/charts/` — Recharts for BP/Glucose.
- `/src/components/dashboard/` — RiskQueue table and Onboarding form.
- `/src/services/api.ts` — Axios instance with auth interceptors.

## 🎨 UX & DESIGN RULES

- **Triage Speed:** Use skeleton loaders for the risk queue. Sorting must be instant.
- **Color Tokens:** HIGH = `#EF4444`, MODERATE = `#F59E0B`, LOW = `#10B981`.
- **Chart Rules:** Always include horizontal reference lines (BP: 120/80, Glucose: 100).
- **Data Density:** Maximize screen space. Use data tables with sticky headers.
- **Accessibility:** Support light and dark mode. Contrast ratio > 4.5:1.
